# Introduction to LDA Topic Modelling

This repository contains the materials for the **Silent Disco: Introduction to LDA Topic Modelling** workshop.

The tutorial was developed for the **Centre for Data, Culture & Society (CDCS)** by **Dr. Aybuke Atalay** and **Dr. Somya Iqbal**.

In this workshop, participants will learn how to perform **topic modelling using the Latent Dirichlet Allocation (LDA)** method. The materials are provided in **both R and Python**, allowing participants to follow the version that best matches their programming experience.

Using the supplied scripts and dataset, participants will walk through the main steps of topic modelling, including:

* text preprocessing
* fitting an LDA model
* evaluating modelling choices (such as selecting the number of topics)
* interpreting topic outputs
* visualising topic modelling results

The scripts include explanations and commented code so that participants can follow the workflow step by step and experiment with different modelling choices.


## Workshop Aims

By the end of this workshop, participants should be able to:

* Understand the basic principles behind **LDA topic modelling**
* Perform **text preprocessing** for topic modelling
* Implement an **LDA model** on a text corpus
* Interpret the **topics generated by the model**
* Evaluate modelling decisions, including selecting an appropriate **number of topics**
* Visualise topic modelling outputs

### Data Files

The workshop uses the **US Presidential Inaugural Addresses** corpus.

In the **R tutorial**, this dataset is available directly through the `quanteda` package:

```r
data_corpus_inaugural
```

For participants following the **Python tutorial**, a CSV version of the dataset is provided in this repository:

```
data/inaugural_speeches.csv
```

This file contains the speech texts along with basic metadata such as the president and year.

### Using the Notebook

The notebook contains paragraphs of explanatory text interspersed with grey cells containing code blocks. To run a code block and see the result:

1. Place your cursor within the cell
2. Click the Run button on the top menu
3. The results of running this code will appear below
4. If the results don't appear immediately, check the icon in the browser tab. An eggtimer icon indicates it is processing the code.
5. It is best to follow the notebook from top to bottom as some code blocks will depend on results from previous cells
6. You can edit code blocks yourself and run them to see the results of your changes


#### Clearing the cells

To clear the results and run the code again you can use the Cell menu on the top menu bar.

1. To clear the results of the current cell: Cell > Current Outputs > Clear
2. To clear the results of all cells: Cell > All Output > Clear
